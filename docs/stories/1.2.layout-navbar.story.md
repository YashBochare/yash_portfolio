# Story 1.2: Implement Global Layout & Responsive Navbar

## Status
Approved

## Story
**As a** Developer,
**I want** A responsive navigation bar and layout container,
**so that** Users can navigate the portfolio and the site has consistent structure

## Acceptance Criteria
1. Shared Button component created
2. Sticky navbar with backdrop blur
3. Hamburger menu for mobile
4. Layout wrapper component
5. Smooth scroll navigation

## Tasks / Subtasks
- [ ] Task 1: Create Reusable Components (AC: 1)
  - [ ] Create `src/components/common/Button.jsx`
  - [ ] Implement variants (primary, outline, ghost) and sizes (sm, md, lg)
  - [ ] Support `onClick` and `href` (anchor vs button) props
  - [ ] Apply hover transitions and active states
- [ ] Task 2: Create Layout Component (AC: 4)
  - [ ] Create `src/components/layout/Layout.jsx`
  - [ ] Accept `children` prop
  - [ ] Setup main container with appropriate max-width and padding
  - [ ] Ensure full viewport height structure (`min-h-screen`)
- [ ] Task 3: Implement Navbar Structure (AC: 2)
  - [ ] Create `src/components/layout/Navbar.jsx`
  - [ ] Use `<header>` semantic tag with `fixed`, `top-0`, `w-full`, `z-50`
  - [ ] Apply `backdrop-blur` and background opacity for glass effect
  - [ ] Add logo/brand name on left
- [ ] Task 4: Implement Navigation Links (AC: 5)
  - [ ] Define nav links data (Home, About, Projects, Contact)
  - [ ] Implement smooth scrolling to section IDs (can use simple `href="#id"` with CSS `scroll-behavior: smooth` in html tags, or a helper function)
  - [ ] Style links with hover effects
- [ ] Task 5: Mobile Navigation (AC: 3)
  - [ ] Implement hamburger menu icon (using SVG or icon lib)
  - [ ] Create mobile menu overlay/dropdown
  - [ ] Manage open/close state with `useState`
  - [ ] Close menu when a link is clicked
  - [ ] Ensure accessible focus management (optional but recommended)

## Dev Notes
- **Component Standard**: Functional components with prop validation (or simple props if TS not strictly enforced, but specs say JS). [Source: docs/front-end-spec.md#components]
- **Styling**: Tailwind utility classes. Use `bg-opacity` or `bg-white/80` for glassmorphism.
- **Responsiveness**: Use Tailwind's `md:` or `lg:` prefixes for desktop styles. Mobile-first approach.
- **Icons**: Use `heroicons` (install if needed) or simple SVGs for the hamburger/close icons to avoid heavy deps if possible, otherwise `lucide-react` or `heroicons` are standard choices. *Decision: Recommend installing `lucide-react` or `heroicons/react` for standard icons.*
- **Scroll Behavior**: Add `html { scroll-behavior: smooth; }` to `index.css` or use a global style.

### Testing
- **Test File Location**: `src/components/layout/__tests__/Navbar.test.jsx` (if testing setup)
- **Standards**: Verify mobile menu toggles correctly. Verify links scroll to sections. Verify sticky behavior on scroll.
