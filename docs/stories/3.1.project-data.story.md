# Story 3.1: Create Project Data Structure

## Status
Done

## Story
**As a** Developer,
**I want** A JSON schema for project data,
**so that** Projects can be managed separately from code

## Acceptance Criteria
1. `projects.json` created with complete schema
2. Schema includes: id, title, category, description, image, technologies, caseStudy
3. Case study structure: problem, solution, tech, result
4. Minimum 2-3 sample projects with placeholder data

## Tasks / Subtasks
- [ ] Task 1: Create Data File (AC: 1, 2, 3)
  - [ ] Create `src/data/projects.json`
  - [ ] Define the schema structure carefully to match AC.
  - [ ] Example structure:
    ```json
    {
      "projects": [
        {
          "id": 1,
          "title": "Example Project",
          "category": "React",
          "description": "...",
          "image": "/assets/projects/project-1.webp",
          "technologies": ["React", "Tailwind"],
          "caseStudy": {
             "problem": "...",
             "solution": "...",
             "tech": "...",
             "result": "..."
          }
        }
      ]
    }
    ```
- [ ] Task 2: Populate Mock Data (AC: 4)
  - [ ] Add 3 mock projects with distinct categories (e.g., 'React', 'Fullstack', 'Design') to test filtering later.
  - [ ] Use placeholder images.
