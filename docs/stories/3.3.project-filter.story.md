# Story 3.3: Implement Project Grid with Filter Logic

## Status
Draft

## Story
**As a** Developer,
**I want** A filterable project grid,
**so that** Visitors can filter projects by category

## Acceptance Criteria
1. Filter pills dynamically generated from project categories
2. Active filter highlighted
3. Grid filters projects on click
4. Responsive grid (1 col mobile, 2 tablet, 3 desktop)
5. Uses ProjectCard component

## Tasks / Subtasks
- [ ] Task 1: Create Projects Section (AC: 4, 5)
  - [ ] Create `src/components/sections/Projects.jsx`
  - [ ] Use `Reveal` wrapper
  - [ ] Layout: `grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-8`
- [ ] Task 2: Implement Filter Logic (AC: 1, 2, 3)
  - [ ] Import data from `projects.json`
  - [ ] Extract unique categories: `['All', ...new Set(data.map(p => p.category))]`
  - [ ] State: `activeFilter` (default 'All')
  - [ ] Filter function: `projects.filter(p => activeFilter === 'All' || p.category === activeFilter)`
- [ ] Task 3: Render Filters
  - [ ] Map categories to buttons (Pills)
  - [ ] Style active vs inactive states
  - [ ] `onClick` updates state

## Dev Notes
- **Animation**: Ideally, use `AnimatePresence` + `LayoutGroup` from Framer Motion for the grid filtering animation, but standard React re-render is acceptable for MVP if complex.
- **Keys**: Ensure correct `key` props for list items.

### Testing
- **Test File Location**: `src/components/sections/__tests__/Projects.test.jsx`
- **Standards**: Verify filtering updates the displayed list.
