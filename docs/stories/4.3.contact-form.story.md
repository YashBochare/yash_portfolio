# Story 4.3: Implement Contact Form with EmailJS

## Status
Done

## Story
**As a** Visitor,
**I want** to send a message directly from the website,
**so that** I don't need to open my email client manually.

## Acceptance Criteria
1. Form Fields: Name (required), Email (required), Message (required).
2. Integration with EmailJS.
3. Form validation (client-side).
4. Loading state during submission.
5. Success/Error feedback message (Toast or inline alert).
6. Security: env variables utilized.

## Tasks / Subtasks
- [ ] Task 1: Setup
  - [ ] Install `emailjs-com` (or use browser fetch if preferred, but SDK is easier).
  - [ ] User must provide: `SERVICE_ID`, `TEMPLATE_ID`, `PUBLIC_KEY` in `.env.local`.
- [ ] Task 2: Create Contact Component
  - [ ] Create `src/components/sections/Contact.jsx`.
  - [ ] Layout: "Get in Touch" text + Form.
  - [ ] Inputs: Styled using Tailwind.
  - [ ] State: `formData`, `isSubmitting`, `status` (success/error).
- [ ] Task 3: Integration logic
  - [ ] `handleSubmit`: Prevent default.
  - [ ] Call `emailjs.sendForm` or `send`.
  - [ ] Handle Promise resolve/reject.
  - [ ] Reset form on success.
- [ ] Task 4: UI Integration
  - [ ] Add to `App.jsx`.
