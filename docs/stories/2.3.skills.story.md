# Story 2.3: Build Skills Grid

## Status
Draft

## Story
**As a** Developer,
**I want** A categorized skills grid,
**so that** Visitors can see my technical capabilities

## Acceptance Criteria
1. `skills.json` data structure created
2. Skills categorized: Languages, Frameworks, Tools
3. Responsive grid layout (1-2-3 columns)
4. Skill cards with clean design
5. Uses Reveal component

## Tasks / Subtasks
- [ ] Task 1: Define Data (AC: 1, 2)
  - [ ] Create `src/data/skills.json`
  - [ ] Structure: `{ languages: [], frameworks: [], tools: [] }`
  - [ ] Populate with initial data
- [ ] Task 2: Create Skills Component (AC: 3, 4, 5)
  - [ ] Create `src/components/sections/Skills.jsx`
  - [ ] Use `Reveal` wrapper
  - [ ] Import data
  - [ ] Map through categories
  - [ ] For each category, render a sub-grid of skills
  - [ ] Skill "Card": Simple pill or box design. Can use icons if available, otherwise text only for now.
- [ ] Task 3: Integration
  - [ ] Add `Skills` to `App.jsx` with `id="skills"` (or part of About, but usually separate)

## Dev Notes
- **Data Driven**: Strictly use the JSON file to populate the UI. [Source: docs/front-end-architecture.md#implementation-patterns]
- **Styling**: `grid-cols-2 md:grid-cols-3 lg:grid-cols-4`.
- **Icons**: If using `lucide-react` or similar, you *can* map string names to components, but it's complex. Text-only or simple SVGs are fine for MVP.

### Testing
- **Test File Location**: `src/components/sections/__tests__/Skills.test.jsx`
- **Standards**: Verify all categories render.
